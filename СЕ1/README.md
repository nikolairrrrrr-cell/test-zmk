# СЕ1 — вход для `pdf-zmk2`

Сюда кладите **исходные** чертежи и документы типа **PDF_ZMK2** (изображения, PDF), которые нужно обработать одним прогоном `pdf-zmk2 full`.

**Как у `pdf-zmk` / База2:**

- В **`СЕ1`** — только то, что сканируем (картинки, PDF).
- В **`data/recognition/`** — артефакты пайплайна:
  - `<slug>.table_x4.jpg` — кроп таблицы (создаётся автоматически);
  - `<slug>.json` — **обязательно** заполняется **по этому кропу** (агент читает картинку). Подставлять данные из чужих эталонных JSON/CSV без чтения кропа **нельзя** — см. `docs/pdf_zmk2-instrukciya.md`.

Готовый JSON **не храним в `СЕ1`** — только в `data/recognition`.

По умолчанию CLI берёт вход из **`СЕ1`**. Другой каталог: `--input-dir …` или **`PDF_ZMK2_INPUT_DIR`**.

```bash
py -m app.cli pdf-zmk2 full --sheet-write
```

Результат в Google Таблице пишется на лист **`СЕ1`** (переименование: переменная **`GOOGLE_SHEET_PDF_ZMK2`**).
