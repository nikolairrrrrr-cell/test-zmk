# Масса (вес опор) — как вы и агент понимаете друг друга

**Тулза `ves`** ищет **массу (вес)** опор по позициям на доверенных сайтах, кэширует результат и может выгрузить таблицу в Google Таблицу на лист **`Масса`**.

Терминал **вам не нужен**: вы даёте только **список позиций** и формулируете задачу в чате — **агент** сам запускает команды, при необходимости обновляет файлы и пишет в таблицу.

---

## Что делаете вы

1. Готовите **список позиций** — по **одной позиции на строку** (как в чертеже, спецификации или вашем учёте). Можно вставить прямо в чат **или** положить/назвать файл так, как попросит агент (часто это `ves/test_positions.txt` или отдельный `.txt` в проекте).
2. Пишете агенту, **что нужно сделать** (см. примеры ниже).

Больше ничего: OAuth Google, `credentials.json`, кэш, команды — **не ваша зона ответственности**.

---

## Что делает агент

- Кладёт или обновляет файл со списком позиций (если вы прислали список в чат — переносит в нужный `.txt`).
- Запускает **`ves run`** с флагами по смыслу задачи:
  - **`--online`** — чтобы реально сходить на сайты (без этого при пустом кэше веса не подтянутся).
  - **`--sheet-write`** — записать результат в Google Таблицу (лист **`Масса`**).
  - **`--force-refresh`** — перескрапить **все** позиции из списка, игнорируя кэш (если вы просите «обновить всё заново»).
  - **`--dry-run`** — посчитать, но **не** трогать таблицу (если вы просите только проверку).
- Ведёт локальный кэш **`ves/weights_db.json`**: при обычном запуске уже найденные позиции не дергаются с сайта без необходимости.
- При необходимости **сбрасывает кэш** только для нужных строк (например, «перепроверь последние пять») и снова запускает тулзу.

Структура таблицы на листе **`Масса`**: колонки по доменам-источникам и **«Примечание»** с пояснением (совпало / один источник / расхождение и т.д.).

---

## Примеры фраз в чат

| Вы пишете | Смысл |
|-----------|--------|
| «Вот список позиций — залей веса в таблицу, лист Масса» | Агент сохраняет список, запускает `ves` с `--online` и `--sheet-write`. |
| «Только проверь по этому списку, в Google не пиши» | Прогон с `--dry-run` или без `--sheet-write`. |
| «Обнови веса по всему списку с нуля» | Запуск с `--force-refresh` (и обычно `--online` + `--sheet-write`). |
| «Перепроверь только эти 5 позиций» | Агент снимает кэш для них и перезапускает обработку. |
| «Добавь к уже существующему файлу позиций ещё строки и дополни таблицу» | Агент дописывает `.txt` и снова гоняет `ves` (новые строки уйдут в интернет, старые могут взяться из кэша). |

---

## Кратко

**Вы:** список позиций (текст) + коротко, что нужно (в таблицу / без таблицы / всё заново / выборочно).  
**Агент:** файл позиций, **`ves`**, кэш **`ves/weights_db.json`**, лист **`Масса`** в Google Таблице — всё на своей стороне.
